# CS-320-Software-Test---Contact-Service-Project
This repository showcases my work in software testing, automation, and quality assurance. It includes a contact service application with corresponding unit tests, demonstrating my ability to design, implement, and validate software based on defined requirements.
## Reflection

### How can I ensure that my code, program, or software is functional and secure?
I ensure my code is functional and secure by implementing thorough unit testing and validating all inputs. Writing test cases helps identify defects early, while enforcing constraints such as input length and null checks prevents invalid data from causing issues. Additionally, following best practices such as encapsulation and exception handling improves reliability and security.

### How do I interpret user needs and incorporate them into a program?
I interpret user needs by carefully reviewing requirements and translating them into functional components. For example, in the contact service project, I implemented constraints such as unique IDs and field length limits based on user requirements. I also use testing to verify that the program behaves as expected from the user’s perspective.

### How do I approach designing software?
I approach software design by breaking down requirements into smaller components and building modular, maintainable code. I focus on clarity and organization, ensuring each class has a single responsibility. I also design with testing in mind, which helps guide development and ensures the software meets requirements effectively.
