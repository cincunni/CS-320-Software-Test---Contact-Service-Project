# CS-320-Software-Test---Contact-Service-Project
This repository showcases my work in software testing, automation, and quality assurance. It includes a contact service application with corresponding unit tests, demonstrating my ability to design, implement, and validate software based on defined requirements.
Portfolio Reflection

Throughout this course, I developed a deeper understanding of software testing, automation, and quality assurance. The projects allowed me to apply unit testing, identify defects, and ensure that software meets both functional and non-functional requirements.

How can I ensure that my code, program, or software is functional and secure?
I ensure functionality and security by implementing thorough unit testing and following best practices such as input validation, exception handling, and code reviews. Writing automated tests, like those in my contact service project, helps verify that each component behaves as expected and prevents regressions. Additionally, I focus on limiting invalid inputs and enforcing constraints to improve reliability and security.

How do I interpret user needs and incorporate them into a program?
I interpret user needs by analyzing requirements and breaking them down into specific, testable features. In my projects, I translated requirements into methods and validation rules, ensuring that the program behaves according to user expectations. Writing test cases also helped me confirm that the implementation aligned with those needs.

How do I approach designing software?
I approach software design by first understanding the problem and then breaking it into smaller, manageable components. I focus on modular design, ensuring each class or function has a single responsibility. I also consider scalability and maintainability by writing clean, readable code and using testing to validate design decisions. This structured approach helps create reliable and adaptable software.
